🔥 Hướng dẫn Setup Firebase

Làm theo 6 bước dưới đây để kích hoạt đầy đủ tính năng PoseAlert

1

Tạo Firebase Project

Vào Firebase Console → nhấn "Create a project" (hoặc "Add project").

  • Đặt tên: PoseAlert (hoặc tùy ý)
  • Google Analytics: có thể bỏ qua (Disable)
  • Nhấn "Create project" → đợi vài giây
2

Bật Authentication

Trong Firebase Console, chọn project vừa tạo → menu trái → Build → Authentication

  • Nhấn "Get started"
  • Tab "Sign-in method" → bật "Email/Password"
  • Bật thêm "Google" → chọn support email → Save
3

Tạo Cloud Firestore

Menu trái → Build → Firestore Database → "Create database"

  • Location: chọn asia-southeast1 (Singapore, gần VN nhất)
  • Rules: chọn "Start in test mode" (cho phép đọc/ghi thoải mái 30 ngày)
  • Nhấn "Create"
⚠️ Test mode cho phép ai cũng đọc/ghi được. Sau khi app hoạt động ổn, nên đổi sang rules bảo mật (xem bước 6).
4

Tạo Realtime Database

Menu trái → Build → Realtime Database → "Create Database"

  • Location: chọn Singapore (asia-southeast1)
  • Rules: chọn "Start in test mode"
  • Nhấn "Enable"
5

Lấy Firebase Config

Vào Project Settings (biểu tượng ⚙️ ở góc trên trái) → tab "General"

  • Cuộn xuống phần "Your apps"
  • Nếu chưa có app → nhấn biểu tượng </> (Web) để tạo
  • Đặt nickname: PoseAlert Web
  • Nhấn "Register app"
  • Firebase sẽ hiện đoạn config → copy toàn bộ

Config sẽ trông giống như thế này:

const firebaseConfig = { apiKey: "AIzaSy...........................", authDomain: "posealert-xxxxx.firebaseapp.com", projectId: "posealert-xxxxx", storageBucket: "posealert-xxxxx.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456", databaseURL: "https://posealert-xxxxx-default-rtdb.asia-southeast1.firebasedatabase.app" };
⚠️ databaseURL có thể không có sẵn trong config. Hãy copy nó từ trang Realtime Database (ở thanh URL phía trên data).
⚡

Paste Config vào đây

Dán đoạn config từ Firebase Console vào ô bên dưới. Tool sẽ tự trích xuất các giá trị cần thiết và tạo nội dung file cho bạn.

6

Mở file js/firebase-init.js và dán config

Mở file js/firebase-init.js trong editor → thay thế các giá trị "YOUR_..." bằng config thật.

Nội dung đã trích xuất (nếu có) sẽ hiện ở trên. Copy và dán vào file.

✅ Sau khi điền config xong, refresh trang index.html — banner cảnh báo sẽ biến mất và tất cả tính năng sẽ hoạt động!
🔒

Firestore Security Rules (tùy chọn, nên làm sau)

Khi app đã hoạt động ổn, vào Firestore → Rules và dán:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Users: chỉ đọc/ghi profile của mình match /users/{userId} { allow read: if request.auth != null; allow write: if request.auth != null && request.auth.uid == userId; } // Sessions: chỉ đọc/ghi phiên của mình match /sessions/{sessionId} { allow read: if request.auth != null; allow create: if request.auth != null && request.resource.data.uid == request.auth.uid; } // Friends: cả 2 user liên quan đều đọc/ghi được match /friends/{friendId} { allow read: if request.auth != null; allow create: if request.auth != null; allow update, delete: if request.auth != null; } // Study Rooms: ai cũng đọc/tạo được, chỉ members mới sửa match /study_rooms/{roomId} { allow read: if request.auth != null; allow create: if request.auth != null; allow update: if request.auth != null; } } }
🔒

Realtime Database Rules (tùy chọn)

Vào Realtime Database → Rules và dán:

{ "rules": { "status": { "$uid": { ".read": true, ".write": "$uid === auth.uid" } }, "chats": { "$roomId": { "messages": { ".read": "auth != null", ".write": "auth != null", ".indexOn": ["sentAt"] }, "typing": { ".read": "auth != null", ".write": "auth != null" } } } } }
